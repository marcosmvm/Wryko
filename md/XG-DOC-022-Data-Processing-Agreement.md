## 1. DEFINITIONS

**1.1** "Applicable Data Protection Laws" means all applicable laws relating to data protection and privacy, including but not limited to:
- EU General Data Protection Regulation (GDPR)
- UK GDPR
- California Consumer Privacy Act (CCPA/CPRA)
- Canada's CASL and PIPEDA
- Other applicable privacy regulations

**1.2** "Personal Data" means any information relating to an identified or identifiable natural person processed under the Agreement.

**1.3** "Data Subject" means the individual to whom Personal Data relates.

**1.4** "Processing" means any operation performed on Personal Data, including collection, storage, use, disclosure, or deletion.

**1.5** "Sub-processor" means any third party engaged by Processor to process Personal Data.

**1.6** "Controller" means the entity that determines the purposes and means of processing.

**1.7** "Processor" means the entity that processes Personal Data on behalf of the Controller.

## 2. SCOPE AND ROLES

### 2.1 Nature of Processing

Processor will process Personal Data on behalf of Controller for the purpose of providing B2B lead generation services as described in the Agreement.

### 2.2 Role Designation

| Party | Role | Responsibility |
|-------|------|----------------|
| Client | Controller/Business | Determines purposes and means of processing |
| XGrowthOS | Processor/Service Provider | Processes data per Controller's instructions |

- Email address verification

- Reply management

- Website visitor identification

## 3. PROCESSOR OBLIGATIONS

### 3.1 Lawful Processing

Processor shall:
- Process Personal Data only on documented instructions from Controller
- Inform Controller if instructions violate Applicable Data Protection Laws
- Not process Personal Data for any purpose other than providing the Services

### 3.2 Confidentiality

Processor shall:
- Ensure personnel processing Personal Data are bound by confidentiality obligations
- Limit access to Personal Data to those who need it for service delivery
- Maintain appropriate access controls

### 3.3 Security Measures

Processor shall implement appropriate technical and organizational measures, including:

| Category | Measures |
|----------|----------|
| **Access Controls** | Role-based access, unique credentials, MFA where applicable |
| **Encryption** | TLS 1.2+ for data in transit, AES-256 for data at rest |
| **Monitoring** | Audit logging, intrusion detection, anomaly monitoring |
| **Physical Security** | Cloud infrastructure with SOC 2 compliance |
| **Personnel** | Background checks, training, confidentiality agreements |
| **Incident Response** | Documented procedures, notification protocols |

### 3.4 Sub-processors

**3.4.1 Authorization**

Controller grants general authorization for Processor to engage Sub-processors, subject to:
- Processor maintaining a list of Sub-processors
- Providing notice of new Sub-processors
- Ensuring Sub-processors are bound by equivalent obligations

**3.4.2 Current Sub-processors**

| Sub-processor | Purpose | Location |
|---------------|---------|----------|
| Instantly.ai | Email sending platform | USA |
| n8n Cloud | Workflow automation | Germany/EU |
| OpenAI | AI content generation | USA |
| Google Cloud | Infrastructure | USA |
| Supabase | Database | USA |

**3.4.3 Changes to Sub-processors**

- Controller will be notified 30 days before new Sub-processor engagement
- Controller may object within 14 days
- If objection cannot be resolved, Controller may terminate affected services

### 3.5 International Transfers

For transfers outside EEA/UK:
- Standard Contractual Clauses (SCCs) apply
- Processor will implement supplementary measures as needed
- Data is primarily processed in the USA

## 5. DATA SUBJECT RIGHTS

### 5.1 Rights Under GDPR

Processor shall assist Controller in responding to Data Subject requests for:
- Access to Personal Data
- Rectification of inaccurate data
- Erasure ("right to be forgotten")
- Restriction of processing
- Data portability
- Objection to processing

### 5.2 Rights Under CCPA

Processor shall assist Controller with:
- Right to know what data is collected
- Right to delete Personal Data
- Right to opt-out of "sales" (Note: XGrowthOS does not sell data)
- Right to non-discrimination

### 5.3 Response Timeline

| Request Type | Response Time |
|--------------|---------------|
| Opt-out / Unsubscribe | Same business day |
| Erasure request | 72 hours |
| Access request | 30 days |
| Rectification | 72 hours |
| Other requests | 30 days |

### 5.4 Request Handling

1. Data Subject contacts Controller or Processor
2. Processor notifies Controller within 24 hours
3. Controller provides instructions
4. Processor executes request per instructions
5. Both parties document completion

---

## 6. DATA BREACH NOTIFICATION

### 6.1 Processor Obligations

In the event of a Personal Data breach, Processor shall:

| Action | Timeline |
|--------|----------|
| Notify Controller | Within 72 hours of discovery |
| Investigate | Immediately |
| Contain breach | As soon as possible |
| Document incident | Within 24 hours |
| Provide detailed report | Within 5 days |

### 6.2 Notification Contents

Breach notification shall include:
- Nature of the breach
- Categories and approximate number of Data Subjects affected
- Categories and approximate number of records affected
- Likely consequences
- Measures taken or proposed

### 6.3 Controller Notification to Authorities

Controller is responsible for notifying supervisory authorities and Data Subjects as required by law.
